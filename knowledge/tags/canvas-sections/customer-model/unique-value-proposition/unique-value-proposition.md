# unique-value-proposition

## Definition
The `unique-value-proposition` tag identifies content about the distinct benefit provided to each customer type, directly addressing their specific jobs to be done and differentiating from existing alternatives.

## Usage Criteria
Apply this tag to content that:
- Defines and explains value proposition development
- Provides frameworks for creating compelling value propositions
- Discusses value proposition testing and validation
- Analyzes value proposition effectiveness

## Always Include When
- Content focuses on value proposition creation and refinement
- Value proposition frameworks and methodologies
- Value proposition testing approaches
- Differentiation and positioning strategies

## Never Include When
- General marketing or messaging content not specific to value propositions
- Product feature lists without value framing
- Benefits statements not tied to customer jobs

## Required Combinations
- Must include appropriate canvas-sections hierarchy: [customer-model, unique-value-proposition]

## Content Examples
- Value proposition canvas methodology
- Value proposition statement templates
- Value proposition testing frameworks
- Customer value validation approaches

## Related Tags
- Parent tags: [customer-model](../customer-model.md)
- Often appears with [customers](../customers/customers.md) (value for specific customer types)
- Often paired with [jobs-to-be-done](../jobs-to-be-done/jobs-to-be-done.md) (addressing specific jobs)
- May combine with [existing-alternatives](../jobs-to-be-done/existing-alternatives/existing-alternatives.md) (differentiation)
- Often appears with [solution](../solution/solution.md) (value delivered through solutions)

## Key Concepts
- Customer-centric value framing
- Job-to-be-done alignment
- Differentiation from alternatives
- Value for different customer types
- Emotional and functional benefits
- Quantifiable value propositions
- Value hypothesis testing
