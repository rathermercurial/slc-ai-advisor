# existing-alternatives

## Definition
The `existing-alternatives` tag identifies content about current solutions or competitors addressing the same jobs that customers are trying to get done, highlighting both market validation and competitive landscape.

## Usage Criteria
Apply this tag to content that:
- Analyzes existing competitive solutions
- Discusses market validation through existing alternatives
- Provides frameworks for competitive analysis
- Explores how customers currently solve the problem

## Always Include When
- Content focuses on competitive landscape analysis
- Current solution identification methodologies
- Market validation through existing alternatives
- Competitive positioning and differentiation

## Never Include When
- Content is about jobs to be done generally (use `jobs-to-be-done`)
- General competitive analysis not tied to JTBD framework
- Market research not focused on alternative solutions

## Required Combinations
- Must include appropriate canvas-sections hierarchy: [customer-model, jobs-to-be-done, existing-alternatives]

## Content Examples
- Competitive analysis frameworks
- Existing solution mapping techniques
- Market validation through alternatives
- Customer current behavior analysis

## Related Tags
- Parent tags: [jobs-to-be-done](../jobs-to-be-done.md), [customer-model](../../customer-model.md)
- Often appears with [unique-value-proposition](../unique-value-proposition/unique-value-proposition.md) (differentiation from alternatives)
- May combine with venture-stage tags for stage-specific competitive analysis

## Key Concepts
- Market validation indicators
- Competitive landscape mapping
- Customer current solutions
- Workarounds and makeshift solutions
- Non-consumption as an alternative
- Direct vs. indirect competitors
