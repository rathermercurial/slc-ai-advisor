# solution

## Definition
The `solution` tag identifies content about the actual product or service that delivers customer value and creates impact - a high-level description of what the enterprise provides to address customer jobs and create social/environmental outcomes.

## Usage Criteria
Apply this tag to content that:
- Defines solution design and development approaches
- Discusses solution-market fit and validation
- Provides frameworks for solution conceptualization
- Analyzes solution effectiveness and iteration

## Always Include When
- Content focuses on solution design and development
- Solution validation and testing methodologies
- Solution evolution and iteration approaches
- Solution-impact alignment strategies

## Never Include When
- Detailed product specifications not tied to canvas framework
- Technical implementation details without strategic context
- Feature lists without value or impact connection

## Required Combinations
- Must include appropriate canvas-sections hierarchy: [customer-model, solution]

## Content Examples
- Solution design frameworks
- Solution validation methodologies
- Solution-impact alignment approaches
- Solution iteration and development strategies

## Related Tags
- Parent tags: [customer-model](../customer-model.md)
- Often appears with [unique-value-proposition](../unique-value-proposition/unique-value-proposition.md) (solution delivers value)
- Often paired with [jobs-to-be-done](../jobs-to-be-done/jobs-to-be-done.md) (solution addresses jobs)
- May combine with impact-model activities (solution creates activities)
- Often appears with venture-stage tags for stage-appropriate solutions
