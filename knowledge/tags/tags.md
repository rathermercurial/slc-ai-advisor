# SLC Knowledge Base Tag System

This directory contains detailed definitions for every tag used in the SLC Knowledge Base. Each tag has its own definition file with usage criteria, examples, and relationships to other tags.

## Tag Structure

The tag system follows the hierarchical structure of the knowledge base:

### [Canvas Section Tags](canvas-section/canvas-section.md)
Core framework components and their sub-elements:
- **Purpose** - Foundational problem statements
- **Customer Model** - Value creation for customers
  - Customers (and Early Adopters)
  - Jobs to be Done (and Existing Alternatives)
  - Unique Value Proposition
  - Solution
  - Assumption Risk
- **Impact Model** - Purpose to outcomes chain
  - Issue, Participants, Activities, Outputs
  - Short/Medium/Long-term Outcomes, Impact
  - Assumption Risk
- **Economic Model** - Financial sustainability
  - Channels, Revenue, Costs, Advantage, Financial Model
  - Assumption Risk
- **Key Metrics** - Cross-model measurement

### [Venture Stage Model Tags](venture-stage-model/venture-stage-model.md)
Tags related to different stages of venture development:
- Idea Stage, Early Stage, Growth Stage, Scale Stage

### [Venture Type Tags](venture-type/venture-type.md)
Comprehensive categorization system for venture characteristics:
- **[Impact Area](venture-type/impact-area/impact-area.md)** - Sectoral and SDG-aligned impact areas (34 tags)
- **[Impact Mechanism](venture-type/impact-mechanism/impact-mechanism.md)** - How ventures create impact (10 tags)
- **[Legal Structure](venture-type/legal-structure/legal-structure.md)** - Organizational forms (11 tags)
- **[Revenue Source](venture-type/revenue-source/revenue-source.md)** - Income generation models (11 tags)
- **[Funding Source](venture-type/funding-source/funding-source.md)** - Capital acquisition methods (7 tags)
- **[Industry](venture-type/industry/industry.md)** - Sector classifications (17 tags)

## Tagging Rules

1. **Hierarchical Tagging**: Include all levels of the hierarchy
   - Example: `[canvas-section, customer-model, customers, early-adopters]`
   - Example: `[venture-type, impact-area, health]`

2. **Required Tags**: Every note must have at least one primary category tag

3. **Consistent Naming**: 
   - Lowercase only
   - Use hyphens for multi-word tags
   - No spaces or special characters

4. **Tag Inheritance**: Sub-category tags should include their parent tags

5. **Multi-dimensional Tagging**: Ventures can be tagged across multiple venture-type dimensions
   - Example: `[venture-type, impact-area, health, legal-structure, nonprofit-inc, revenue-source, service-fees]`

## Usage Guidelines

- **Before adding tags**: Check the relevant definition file
- **When in doubt**: Err on the side of more specific tags
- **For new content**: Follow the hierarchical pattern
- **For cross-cutting content**: Use multiple category tags appropriately
- **For venture types**: Use multiple dimensions to fully characterize ventures
